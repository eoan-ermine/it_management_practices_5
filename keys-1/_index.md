## **Описание**

Заказчик хочет создать мобильное приложение для отслеживания пунктов раздельного сбора мусора в городе. Основные функции: карта с отметками пунктов приёма, информация о типах принимаемого сырья, уведомления об акциях, личный кабинет с историей сдач.

Срок -- 4 месяца. Бюджет ограничен.

Команда: 2 разработчика, 1 дизайнер, 1 маркетолог.

## **Задание**

Разработайте СДР для этого проекта, используя **продуктовый подход**.

Уровень детализации -- до пакетов работ, за которые может отвечать один специалист.

### **Что нужно сделать**

1. Определите конечный продукт и его основные компоненты (например: интерфейс, база данных, карта, система уведомлений и т.д.).

2. Проведите декомпозицию до уровня элементарных работ (например: «создать макет экрана карты», «настроить API для геоданных»).

3. Оформите СДР в виде иерархической схемы или таблицы.

4. Проверьте себя по [чек-листу](./../chek-list).
